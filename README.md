TOCA Player Portal
Overview
Build a full-stack web application that allows players to see their past and future training sessions. The application should be built as a single page application (SPA) that includes a React-based frontend and a Node.js backend.
Features & Requirements
“Sign in” screen that allows a player to enter their email address. This is not a true sign in/authentication screen. Simply collect and use the entered email address.
Header showing a logo and a “Logout” button that brings the player back to the “Sign in” screen.
A navigation menu with the following items:
Home
About TOCA
Profile
Home screen that lists the player’s past training sessions and future appointments. Past training sessions should be clickable and bring the player to a session details screen.
“About TOCA” screen that shows some general information about TOCA and how it helps players to improve their soccer skills. Feel free to take some content from the TOCA website.
Profile screen that shows the player’s email address and other profile information.
Out of Scope
Sign up, Sign in, Authentication
Use of a database is optional. Data can simply be stored and read from JSON files.

Other Notes
You are allowed and encouraged to use any AI tools of your choice.
When using AI to generate code, you should still understand the code and be able to explain what it is doing.
You may use other libraries than the ones listed above (must use required ones though).
Use your best judgement as to how to lay out and present information.
